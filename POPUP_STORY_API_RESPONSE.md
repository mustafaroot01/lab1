# دليل ربط واجهات إعلانات الستوري (Popup Stories API Guide) 📱✨

يوفر هذا الدليل التفاصيل الكاملة لمبرمج الموبايل لربط ميزة "إعلانات الستوري الطولية" (Popup Stories) التي تظهر في شاشة الهوم للتطبيق. 

يحتوي النظام على **3 مسارات (APIs)** أساسية للتعامل مع الستوريات وتتبع الإحصائيات (المشاهدات والنقرات)، بالإضافة إلى تفصيل حالتي الزر (CTA) المتوفرتين.

---

## أولاً: المسارات الأساسية (API Endpoints)

### 1. جلب الستوريات النشطة (Fetch Active Stories)
هذا المسار يقوم بإرجاع جميع الستوريات المفعلة حالياً والمجدولة للظهور، مرتبة حسب الترتيب (Sort Order) الذي حددته الإدارة.

* **المسار (URL):** `GET /api/popup-stories/active`
* **نوع الطلب:** `GET`
* **المصادقة (Auth):** لا يحتاج إلى توكن (متاح كزائر أو مستخدم مسجل).
* **شكل الاستجابة (JSON Response):**

```json
{
  "status": true,
  "message": "تم جلب إعلانات الستوري النشطة بنجاح",
  "stories": [
    {
      "id": 15,
      "title": "عروض باقات الفيتامينات الشاملة",
      "image_url": "https://lab.diyala.org/storage/popups/vitamin_offer.png",
      "image_path": "popups/vitamin_offer.png",
      "duration_seconds": 7,
      "display_frequency": "once_per_day",
      "display_frequency_label": "مرة واحدة يومياً للمراجع",
      "button_text": "احجز الباقة الآن",
      "button_link_type": "link",
      "button_link_target": "https://lab.diyala.org/packages/12",
      "start_at": "2026-07-20 00:00:00",
      "end_at": "2026-07-30 23:59:59",
      "sort_order": 1,
      "is_active": true,
      "is_currently_active": true,
      "views_count": 1500,
      "clicks_count": 340,
      "ctr_percentage": 22.6,
      "created_at": "2026-07-15 14:30"
    }
  ]
}
```

#### 💡 ملاحظات هامة حول حقل تكرار الظهور (`display_frequency`):
يجب على تطبيق الموبايل احترام هذا الحقل لكي لا ينزعج المستخدم:
1. `always`: اظهر الستوري في كل مرة يفتح فيها المستخدم شاشة الهوم.
2. `once_per_session`: اظهر الستوري مرة واحدة فقط طوال فترة بقاء التطبيق مفتوحاً (تختفي إذا تنقل بين الشاشات وعاد للهوم).
3. `once_per_day`: اظهر الستوري مرة واحدة فقط كل 24 ساعة لهذا المستخدم (يُفضل حفظ حالة الظهور محلياً في SharedPreferences / UserDefaults).

---

### 2. تسجيل مشاهدة للستوري (Record a View)
لكي تظهر الإحصائيات للإدارة بشكل دقيق (معدل الوصول)، يجب على تطبيق الموبايل إرسال طلب صامت (Background Request) عندما ينتهي المستخدم من مشاهدة الستوري أو تظهر له.

* **المسار (URL):** `POST /api/popup-stories/{id}/view`
* **نوع الطلب:** `POST`
* **المتغيرات:** استبدل `{id}` برقم الستوري.
* **شكل الاستجابة (JSON Response):**

```json
{
  "status": true
}
```

---

### 3. تسجيل نقرة على زر الستوري (Record a Click)
عندما يضغط المستخدم على الزر (Call-to-Action) الموجود في الستوري لكي يتوجه إلى باقة أو رابط، يجب إرسال هذا الطلب لحساب نسبة النقر للظهور (CTR).

* **المسار (URL):** `POST /api/popup-stories/{id}/click`
* **نوع الطلب:** `POST`
* **المتغيرات:** استبدل `{id}` برقم الستوري.
* **شكل الاستجابة (JSON Response):**

```json
{
  "status": true
}
```

---

## ثانياً: حالات الزر للستوري (CTA Button States) 🎯

الإدارة في لوحة التحكم تقوم بإضافة الستوري إما كـ "إعلان صوري فقط" أو "إعلان تفاعلي مع رابط". حقل `button_link_type` يخبر الموبايل بالحالة:

### الحالة الأولى: بدون رابط / صورة فقط (`none`)
الستوري عبارة عن إعلان صوري فقط للإخبار (مثل تهنئة بالعيد أو إعلان لا يتطلب تفاعلاً).
* **الإجراء البرمجي للموبايل:** لا تظهر أي زر أسفل الستوري، واكتفِ بعرض الصورة لمدة `duration_seconds` ثم أغلق الستوري.
* **البيانات المستلمة في هذه الحالة:**
```json
{
  "button_link_type": "none",
  "button_text": null,
  "button_link_target": null
}
```

### الحالة الثانية: إعلان يحتوي على زر ورابط (`link`)
الستوري يوجه المستخدم إلى شاشة معينة أو موقع أو باقة. الإدارة ستضع نص الزر والرابط المطلوب.
* **الإجراء البرمجي للموبايل:** اعرض الزر باستخدام النص الموجود في `button_text`. وعند الضغط عليه، استخدم دالة فتح الروابط بناءً على الرابط الموجود في `button_link_target`.
* **البيانات المستلمة في هذه الحالة:**
```json
{
  "button_link_type": "link",
  "button_text": "شاهد الباقات والتفاصيل",
  "button_link_target": "https://..."
}
```

---
**ملاحظة أخيرة لمبرمج الموبايل:** لا تنسَ إرسال طلب الـ `click` (المسار الثالث) **قبل** أن تفتح الرابط (Link) لضمان تسجيل النقرة في إحصائيات الإدارة.
