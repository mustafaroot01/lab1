# دليل اختبار واجهات تطبيق الموبايل (Mobile API Complete Testing Guide)
**المشروع:** نظام إدارة المختبرات والزيارات المنزلية (Laravel 11 + Sanctum + Mobile API)  
**الهدف:** توثيق جميع نقاط الاتصال (Endpoints) الخاصة بتطبيق المريض، وطريقة اختبارها، والمشاكل الشائعة (Edge Cases) وكيفية التعامل معها.

---

## 1. إعدادات أساسية لطلب الـ API (Base Headers & URL)
* **الرابط الأساسي (Base URL):**  
  `http://127.0.0.1:8001/api` (أو رابط السيرفر المباشر)
* **الترويسات الأساسية المطلوبة لجميع الطلبات (Headers):**
  ```http
  Accept: application/json
  Content-Type: application/json
  ```
* **في الطلبات التي تتطلب تسجيل دخول (Auth Required):**
  ```http
  Authorization: Bearer {USER_SANCTUM_TOKEN}
  ```

---

## 2. نظام المصادقة والتسجيل المباشر برمز التحقق (Passwordless OTP Auth)

### 2.1 إرسال رمز التحقق OTP
* **الرابط:** `POST /api/mobile/auth/send-otp`
* **صلاحية الدخول:** متاح للعامة (Public)
* **جسم الطلب (Body):**
  ```json
  {
    "phone": "07700000000"
  }
  ```
* **الرد الناجح (200 OK):**
  ```json
  {
    "status": true,
    "message": "تم إرسال رمز التحقق بنجاح",
    "dev_otp": "1234" // يظهر في وضع التطوير للتجربة السريعة
  }
  ```
* **⚠️ مشاكل وحالات خاصة يجب الحذر منها (Edge Cases):**
  - **رقم هاتف غير صالح:** يجب التأكد من أن الموبايل يرسل أرقام العراق بالصيغة الصحيحة (مثلاً يبدأ بـ `07` أو `+9647`).
  - **الضغط المتكرر (Rate Limiting):** في حال ضغط المستخدم على زر "إرسال الكود" عدة مرات متتالية، يجب تعطيل الزر لمدة 30 ثانية في واجهة الموبايل لمنع الحظر.

---

### 2.2 التحقق من كود الـ OTP والحصول على التوكن
* **الرابط:** `POST /api/mobile/auth/verify-otp`
* **صلاحية الدخول:** متاح للعامة (Public)
* **جسم الطلب (Body):**
  ```json
  {
    "phone": "07700000000",
    "otp": "1234"
  }
  ```
* **الرد الناجح (200 OK):**
  ```json
  {
    "status": true,
    "message": "تم التحقق بنجاح",
    "token": "1|abcdef123456789...",
    "user": {
      "id": 1,
      "phone": "07700000000",
      "name": "أحمد محمد",
      "is_profile_completed": true
    }
  }
  ```
* **⚠️ نقاط انتباه لمبرمج الموبايل:**
  - إذا كان `is_profile_completed: false` يجب توجيه المريض فوراً إلى **شاشة إكمال الملف الشخصي** (الاسم والتاريخ والجنس).
  - يجب حفظ قيم الـ `token` في الحفظ الآمن للتطبيق (`SecureStorage` في Flutter أو `KeyChain` في iOS).

---

### 2.3 إكمال الملف الشخصي للمريض الجديد
* **الرابط:** `POST /api/mobile/auth/complete-profile`
* **صلاحية الدخول:** يتطلب توكن (`Authorization: Bearer ...`)
* **جسم الطلب (Body):**
  ```json
  {
    "name": "أحمد محمد علي",
    "gender": "male",
    "birth_date": "1990-05-15"
  }
  ```

---

### 2.4 تسجيل الخروج وطلب حذف الحساب (Mandatory for App Store)
* **تسجيل الخروج (إلغاء التوكن الحالي):**  
  `POST /api/mobile/auth/logout` (يتطلب Sanctum Token)
* **إرسال طلب حذف الحساب (مطلوب لقبول تطبيقك في Apple App Store):**  
  `DELETE /api/mobile/auth/delete-account` (يتطلب Sanctum Token)  
  يرسل طلب حذف للإدارة مع عدم حذف الحساب فعلياً وإرجاع الرسالة:  
  *"تم إرسال طلب حذف الحساب بنجاح، سيتم مراجعة طلبك من قبل الإدارة."*

## 3. الشاشة الرئيسية والكتالوج والعروض (Home & Catalog)

### 3.1 جلب البنرات الإعلانية (Banners)
* **الرابط:** `GET /api/banners`
* **الرد الناجح:** قائمة البنرات المفعلة وصورها وروابطها الترويجية.
* **⚠️ مشكلة شائعة وكيفية حلها:**
  - **حجم الصور وسرعة التحميل:** تأكد أن روابط الصور الراجعة (`image_url`) كاملة (`http://.../storage/...`) واستخدم كاش للصور (`CachedNetworkImage` في Flutter) لتجنب إعادة تحميلها في كل مرة يفتح فيها التطبيق.

---

### 3.2 جلب كتالوج التحاليل والمجموعات الطبية
* **الرابط:** `GET /api/mobile/catalog?search=`
* **الرد الناجح:**
  ```json
  {
    "status": true,
    "groups": [
      {
        "id": 1,
        "name_ar": "محتويات الدم والتخثر",
        "icon": "droplet",
        "color": "#fa0400",
        "tests_count": 5,
        "tests": [
          {
            "id": 1,
            "item_type": "test",
            "name_ar": "صورة الدم الكاملة CBC",
            "price": 22000,
            "fasting_required": false,
            "result_time": "15 دقيقة",
            "description": "وصف التحليل..."
          }
        ]
      }
    ],
    "packages": [ ... ]
  }
  ```

---

### 3.3 قائمة الباقات وتفاصيل الباقة (Packages)
* **قائمة الباقات:** `GET /api/mobile/packages`  
  يرجع قائمة الباقات مع حساب `discount_percent` (نسبة الخصم) وقائمة التحاليل المشمولة.
* **تفاصيل باقة محددة:** `GET /api/mobile/packages/{id}`
* **تفاصيل تحليل محدد:** `GET /api/mobile/tests/{id}`
* **⚠️ معالجة الخطأ 404:** في حال كان التحليل أو الباقة غير مفعل أو تم حذفه، سيرجع السيرفر `404` مع رسالة وضوح يجب عرضها للمريض.

---

## 4. الفروع والتغطية الجغرافية (Coverage & Addresses)
* **جلب المحافظات والمناطق المخدومة:** `GET /api/governorates`
* **⚠️ مشكلة شائعة (إلغاء واختيار منطقة جديدة):**
  - المريض يمكنه اختيار قضاء أو منطقة معينة، أو ترك اختيار المنطقة ليتعامل معه الفرع الرئيسي.
  - يجب على التطبيق عدم إجبار المريض على تحديد القضاء إذا أراد طلب فحص داخل الفرع أو إذا كانت المنطقة عامة.

---

## 5. السلة والكوبونات والوصفات الطبية (Cart & Prescription)

### 5.1 معاينة السلة وحساب التكلفة
* **الرابط:** `POST /api/mobile/cart/preview`
* **جسم الطلب:**
  ```json
  {
    "items": [
      { "type": "test", "id": 1 },
      { "type": "package", "id": 1 }
    ]
  }
  ```
* **الرد:** الإجمالي، أسعار التحاليل، وسعر الخدمة المنزلية إن وجد.

---

### 5.2 التحقق من كوبون الخصم
* **الرابط:** `POST /api/mobile/coupon/validate`
* **جسم الطلب:**
  ```json
  {
    "code": "HEALTH2026",
    "total_amount": 50000
  }
  ```

---

### 5.3 رفع صورة الروشتة الطبية (Prescription / Referral Upload)
* **الرابط:** `POST /api/mobile/orders/upload-referral`
* **نوع الطلب:** `multipart/form-data`
* **الحقل المطلوب:** `image` (ملف صورة JPG/PNG).
* **⚠️ مشكلة شائعة في الموبايل:** يجب ضغط الصورة قبل رفعها من الهاتف حتى لا يستغرق الرفع وقتاً طويلاً على شبكة 4G.

---

## 6. إنشاء الطلبات وإدارة طلباتي (Orders)

### 6.1 إنشاء طلب فحص جديد
* **الرابط:** `POST /api/mobile/orders` (يتطلب Sanctum Token)
* **جسم الطلب:**
  ```json
  {
    "service_type": "home_visit",
    "items": [
      { "type": "test", "id": 1 }
    ],
    "visit_date": "2026-07-12",
    "visit_time": "10:00 AM",
    "address_details": "بغداد - الكرادة - شارع 62",
    "notes": "يرجى الاتصال قبل الوصول"
  }
  ```

---

### 6.2 جلب قائمة طلباتي، تفاصيل الطلب، وسجل الحالات
* **قائمة طلباتي (`My Orders`):** `GET /api/mobile/orders`
* **تفاصيل طلب وسجل الحالات والنتائج:** `GET /api/mobile/orders/{id}`  
  يرجع تفاصيل الطلب كاملة ويتضمن:
  - **ملفات النتيجة المخبرية (`results`):** روابط تحميل أو عرض ملفات الـ PDF أو الصور.
  - **سجل تتبع الحالات (`status_logs`):** التسلسل الزمني الكامل من إنشاء الطلب حتى صدور النتيجة.
  *(ملاحظة: هذا السجل الكامل يظهر أيضاً لموظف المختبر في لوحة التحكم داخل ملف المريض البروفايل `GET /api/patients/{id}`).*
* **إلغاء طلب:** `POST /api/mobile/orders/{id}/cancel`

---

## 7. إدارة السجل الطبي للمريض (Medical Records)
* **جلب السجل الشامل:** `GET /api/mobile/medical-records`
* **إضافة مرض / دواء / حساسية:** `POST /api/mobile/medical-records`
  ```json
  {
    "type": "chronic_disease", // أو medication أو allergy
    "disease_name": "السكري النوع الثاني",
    "severity": "medium"
  }
  ```
* **تعديل عنصر:** `PUT /api/mobile/medical-records/{type}/{id}`
* **حذف عنصر:** `DELETE /api/mobile/medical-records/{type}/{id}`

---

## 8. الصفحات القانونية، الأسئلة الشائعة ومعلومات التواصل (Legal, FAQ & Contact)
جميع هذه المسارات متاحة للعامة (Public) ولا تتطلب توكن تسجيل دخول:

### 8.1 الشروط والأحكام وسياسة الخصوصية (Legal Pages)
* **الرابط:** `GET /api/legal-pages`
* **الرد الناجح:** قائمة الصفحات مع المحتوى (`slug: terms` للشروط والأحكام، و `slug: privacy` لسياسة الخصوصية).

---

### 8.2 الأسئلة الشائعة مصنفة (FAQs)
* **الرابط:** `GET /api/faqs`
* **الرد الناجح:** قائمة الأسئلة والأجوبة مرتبة حسب التصنيف (`category`).

---

### 8.3 معلومات التواصل وأرقام الدعم (Contact Info)
* **الرابط:** `GET /api/contact-infos`
* **الرد الناجح:** أرقام الهاتف، واتساب الدعم، روابط شبكات التواصل الاجتماعي، والعنوان المعتمد.

---

## 9. قائمة المراجعة لاختبار التطبيق قبل الإطلاق (Pre-Launch Checklist)
1. ✅ **انتهاء صلاحية التوكن (Token Expiration):** تأكد أنه في حال رجوع السيرفر بـ `401 Unauthorized` يقوم التطبيق بتسجيل خروج سلس للمستخدم وإظهار شاشة إدخال رقم الموبايل.
2. ✅ **انقطاع الإنترنت (Offline Handling):** إظهار رسالة لطيفة للمريض عند انقطاع الاتصال بدلاً من شاشة بيضاء.
3. ✅ **تنسيق التواريخ والخط العربي:** التأكد من دعم الخط العربي (Cairo) في واجهات الجوال وتنسيق الأرقام والأسعار بوضوح (مثلاً `25,000 د.ع`).
